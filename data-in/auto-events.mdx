## Overview

**Auto-events are off by default.** They run only when you either:

- Set **`auto-events="true"`** on the analytics script tag, or  
- Call **`Cryptique.enableAutoEvents()`** after load.

When enabled, the SDK sends named **auto events** through the same pipeline as `Cryptique.track()`, including default properties and (when configured) [group](/data-in/sdk-reference/groups) enrichment.

<Warning>
  If auto-events are disabled, **`trackAutoEvent` does not send** any of the events below — including error, performance, and page summary events. They all share the same global switch and filters.
</Warning>

---

## Gating and filters

Three layers apply before an auto event is sent:

1. **Global:** `CONFIG.AUTO_EVENTS.enabled` (script attribute or `enableAutoEvents` / `disableAutoEvents`).
2. **Path:** `auto-events-disabled-paths` or `Cryptique.setAutoEventsDisabledPaths()` — prefix match on `pathname` (e.g. `/admin` matches `/admin` and `/admin/users`).
3. **Per event:** `auto-events-disabled-events` or `Cryptique.disableAutoEvent('event_name')` — names are compared **case-insensitively**.

**SPA navigation:** On `pushState`, `replaceState`, and `popstate`, the SDK re-evaluates path rules. If the new route allows auto-events and setup has not run, listeners are attached.

## JavaScript API

| Method | Purpose |
|--------|---------|
| `Cryptique.enableAutoEvents()` | Turn on auto-events and run setup if the current path allows it. |
| `Cryptique.disableAutoEvents()` | Turn off all auto event sending. |
| `Cryptique.setAutoEventsDisabledPaths(paths)` | Set path exclusions (string or array of strings). |
| `Cryptique.disableAutoEvent(events)` | Add event name(s) to the disabled list. |
| `Cryptique.enableAutoEvent(events)` | Remove event name(s) from the disabled list. |
| `Cryptique.setAutoEventsDisabledEvents(events)` | Replace the entire disabled-event list. |
| `Cryptique.getAvailableAutoEvents()` | Returns the **primary interaction** event names (subset; see note below). |
| `Cryptique.getAutoEventsConfig()` | Returns `{ enabled, disabledPaths, disabledEvents, availableEvents, currentPath, shouldTrack }`. |

<Note>
  **`getAvailableAutoEvents()`** currently returns: `page_view`, `element_click`, `rage_click`, `dead_click`, `page_scroll`, `form_submit`, `form_focus`, `media_play`, `media_pause`, `text_selection`.  
  Other auto events in the table below (e.g. `js_error`, `page_summary`) are still real events and can be **disabled by name** via `auto-events-disabled-events` or `disableAutoEvent`, even though they are not in that array.  
  **`form_focus`** is listed for API symmetry; per-field focus is summarized in aggregates (e.g. `page_summary`) rather than emitted as its own high-volume auto event — see [Forms](#forms).
</Note>

## Event catalog (implemented names)

Use these **exact** `event_name` values in dashboards, filters, and `auto-events-disabled-events`.

### Navigation & page

| Event | When it fires | Notes |
|-------|----------------|-------|
| `page_view` | After load (short delay) for the current route | Includes timing hints such as `page_load_time`, `scroll_position`. SPA route changes are covered by the tracker setup + navigation hooks. |

### Clicks & scroll

| Event | When it fires | Notes |
|-------|----------------|-------|
| `element_click` | Every click | Includes viewport and page coordinates, document dimensions, `double_click`, element metadata (`element_tag_name`, `element_id`, `element_text`, etc.). |
| `rage_click` | ≥ 3 clicks on the **same** element within 1 s, debounced 800 ms after the last click | Sends final `click_count`, `time_span`, scroll/page context. |
| `dead_click` | Click on a **non-interactive** element, and after 800 ms the URL and DOM mutation snapshot are unchanged | Reduces false positives when SPAs update the DOM without navigation. |
| `page_scroll` | Scroll position updates (debounced); fires when **max** scroll depth increases | Properties include `scroll_depth`, `max_scroll_reached`, `scroll_position`, document size. |

### Forms

| Event | When it fires | Notes |
|-------|----------------|-------|
| `form_submit` | `submit` on a `<form>` | Captures form id, action, method, positions — **not** field values. |
| `form_validation_error` | Browser `invalid` event on inputs / textareas | Field metadata and `validation_message` only. |
| `form_abandoned` | User focused / filled fields then leaves without submit (`beforeunload`) | Counts filled fields, last field, time on form. |

<Tip>
  **Per-keystroke** focus/blur/change events are **not** emitted as separate auto-events (to avoid noise). Attention and field timing appear in **`page_summary`** (e.g. `field_dwells`).
</Tip>

### Media

| Event | When it fires | Notes |
|-------|----------------|-------|
| `media_play` | `play` on `<video>` or `<audio>` | Throttled per element (~2 s) to reduce scrubbing noise. |
| `media_pause` | `pause` on `<video>` or `<audio>` | Same throttling. |

### Selection & clipboard

| Event | When it fires | Notes |
|-------|----------------|-------|
| `text_selection` | User selects text (debounced); meaningful length ≥ 3 chars, 1 s cooldown | Selected text capped (e.g. 500 chars in payload); includes page and element context. |
| `copy_action` | Native `copy` | Does not include full copied text by default; includes length and coarse element info. |
| `context_menu` | Right-click (`contextmenu`) | Element tag/id/class and coordinates. |

### Errors & network

| Event | When it fires | Notes |
|-------|----------------|-------|
| `js_error` | `window.error` and unhandled promise rejections | Deduplicated (~10 s). Includes `error_class`, `last_event_name` (last non-error auto event). |
| `network_error` | Fetch/XHR failures or HTTP ≥ 400 for **same-origin** `fetch` (and configured SDK XHR targets) | Cross-origin requests are intentionally not wrapped. |

### Performance

| Event | When it fires | Notes |
|-------|----------------|-------|
| `page_performance` | After `load` + delay | Core Web Vitals–style metrics where available: e.g. LCP, CLS, INP, FCP, TTFB, slowest resources. |

### Engagement summary

| Event | When it fires | Notes |
|-------|----------------|-------|
| `exit_intent` | Mouse leaves viewport **top edge** (debounced ~2 s) | Scroll depth and time on page. |
| `page_summary` | `pagehide` or `visibilitychange` (tab hidden) | Single aggregate: movement/hover grids, tab switches, scroll reversals, field dwells, viewport element dwells, CLS/long tasks/INP, exit classification, etc. |

### Not emitted in current builds

| Name | Status |
|------|--------|
| `element_view` | **Disabled** — individual visibility events were replaced by **`viewport_element_dwells`** (and related fields) inside `page_summary`. |
| `session_start` / `session_end` | **Not** sent as separate auto events in the SDK (session boundaries are handled via session cookies / backend). |
| `media_ended` | **Not** emitted (play/pause only). |
| `element_hover` | **Not** emitted as its own auto event; hover signal is aggregated in `page_summary`. |

Wallet connect/disconnect are **not** auto-events in this SDK — use **`Cryptique.walletAddress()`** and/or **`Cryptique.track()`** for wallet lifecycle.

## Configuration examples

### Script tag

```html
<script>
  var script = document.createElement('script');
  script.src = 'https://cdn.cryptique.io/scripts/analytics/1.0.1/cryptique.script.min.js';
  script.setAttribute('site-id', 'YOUR_SITE_ID');
  script.setAttribute('auto-events', 'true');
  script.setAttribute('auto-events-disabled-paths', '/admin,/internal');
  script.setAttribute('auto-events-disabled-events', 'text_selection,copy_action,page_performance');
  document.head.appendChild(script);
</script>
```

### Runtime

```javascript
Cryptique.enableAutoEvents();
Cryptique.setAutoEventsDisabledPaths(['/admin', '/debug']);
Cryptique.disableAutoEvent(['rage_click', 'dead_click']);
// Inspect state
console.log(Cryptique.getAutoEventsConfig());
```

### Path matching

- `/admin` matches `/admin` and `/admin/users`.
- Trailing slashes are normalized for comparison.

## Privacy and volume

- **Forms:** Submits and validation capture **structure**, not user input values.  
- **Text selection:** A **substring** of selected text may be included (length-capped). Disable `text_selection` if that is too sensitive for your policy.  
- **Clicks:** `element_text` is truncated (e.g. 100 chars).  
- **Volume:** `page_summary` consolidates high-frequency signals into **one** event per exit/hide instead of streaming every mouse move.

## Data on every auto event

Auto events receive the same default context as custom events (session, device, URL, geo when available, UTM, etc.). See [Default Properties](/data-in/default-properties).
