## Cryptique.init()

Initialize the Cryptique SDK with your configuration options. This must be called before using any other Cryptique methods.

```javascript
Cryptique.init(options)
```

### Parameters

<ParamField path="options" type="object" required>
  Configuration object for the SDK.

  <Expandable title="options properties">
    <ParamField path="siteId" type="string" required>
      Your unique Cryptique Site ID. Find this in your dashboard under **Settings â†’ Sites**.
    </ParamField>

    <ParamField path="autoEvents" type="boolean" default="true">
      Enable or disable automatic event tracking. When `true`, Cryptique automatically tracks page views, clicks, scrolls, and other user interactions.
    </ParamField>

    <ParamField path="disabledPaths" type="string[]" default="[]">
      Array of URL paths where auto-events should be disabled. Useful for excluding admin pages, internal tools, or sensitive areas.
      
      ```javascript
      disabledPaths: ['/admin', '/internal', '/settings']
      ```
    </ParamField>

    <ParamField path="disabledEvents" type="string[]" default="[]">
      Array of auto-event names to disable. See [Auto-Tracked Events](/guide/auto-events) for the full list.
      
      ```javascript
      disabledEvents: ['page_scroll', 'text_selection']
      ```
    </ParamField>
  </Expandable>
</ParamField>

## Configuration Examples

### Basic Initialization

```javascript
import Cryptique from 'cryptique-sdk';

Cryptique.init({
  siteId: 'your-site-id'
});
```

### Full Configuration

```javascript
import Cryptique from 'cryptique-sdk';

Cryptique.init({
  siteId: 'your-site-id',
  autoEvents: true,
  disabledPaths: [
    '/admin',
    '/internal',
    '/debug'
  ],
  disabledEvents: [
    'page_scroll',
    'text_selection',
    'media_play',
    'media_pause'
  ]
});
```

### Disable All Auto-Events

For manual-only tracking:

```javascript
Cryptique.init({
  siteId: 'your-site-id',
  autoEvents: false
});

// Now only explicit Cryptique.track() calls will send data
Cryptique.track('Custom Event', { key: 'value' });
```

## Runtime Configuration

You can modify auto-event settings after initialization using these methods:

### Disable Specific Auto-Events

```javascript
// Disable one event
Cryptique.disableAutoEvent('page_scroll');

// Disable multiple events
Cryptique.disableAutoEvent(['page_scroll', 'text_selection']);
```

### Re-Enable Auto-Events

```javascript
// Re-enable one event
Cryptique.enableAutoEvent('page_scroll');

// Re-enable multiple events
Cryptique.enableAutoEvent(['page_scroll', 'text_selection']);
```

### Replace Disabled Events List

```javascript
// Replace the entire disabled events list
Cryptique.setAutoEventsDisabledEvents(['rage_click', 'dead_click']);
```

## Finding Your Site ID

<Steps>
  <Step title="Log in to your dashboard">
    Go to [app.cryptique.io](https://app.cryptique.io) and sign in.
  </Step>
  
  <Step title="Navigate to Settings">
    Click on **Settings** in the sidebar.
  </Step>
  
  <Step title="Copy your Site ID">
    Find your Site ID under the **Sites** section. Click to copy.
  </Step>
</Steps>

<Warning>
  Keep your Site ID secure. While it's not a secret, you shouldn't share it publicly as it could allow others to send events to your account.
</Warning>

## Session Behavior

Cryptique automatically manages user sessions:

| Behavior | Value |
|----------|-------|
| **Session Timeout** | 30 minutes of inactivity |
| **Session ID** | Auto-generated unique identifier |
| **Cross-Tab** | Sessions are shared across tabs on the same domain |
| **Persistence** | Session data persists in browser storage |

A new session starts when:
- User first visits your site
- 30 minutes pass without any user activity
- User clears browser data
- User explicitly calls `Cryptique.reset()`
