## Overview

This page documents all data constraints and limits in the Cryptique SDK. Understanding these limits helps you design your tracking implementation correctly.

---

## Event Limits

### Event Name

| Constraint | Limit |
|-----------|-------|
| Maximum length | 255 characters |
| Allowed characters | Any (stored as-is) |
| Case sensitivity | Yes (case-sensitive) |

```javascript
// ✅ Good
Cryptique.track('Swap Completed');
Cryptique.track('NFT_Minted_Successfully');

// ❌ Too long (255+ chars)
Cryptique.track('This is an extremely long event name that exceeds the maximum allowed character limit...');
```

---

### Event Properties

Event properties (`custom_properties`) are stored as JSONB with minimal validation:

| Constraint | Limit |
|-----------|-------|
| Max properties per event | Unlimited (PostgreSQL JSONB limits apply) |
| Max property key length | No enforced limit |
| Max property value size | No enforced limit |
| Supported types | string, number, boolean, array, object, null |

<Info>
  Event properties have fewer restrictions than user (people) properties. However, keep payloads reasonable for performance.
</Info>

### Reserved Event Properties

These property names are **automatically stripped** from event data (they're captured by the system):

```javascript
// ❌ Don't use these - they'll be removed
{
  page,
  full_url,
  current_url,
  uri,
  referrer,
  user_agent,
  language,
  screen_width,
  screen_height,
  viewport_width,
  viewport_height,
  dpi,
  timestamp,
  page_title,
  title,
  url,
  href
}
```

Use different property names:

```javascript
// ✅ Good alternatives
Cryptique.track('Link Clicked', {
  target_url: 'https://example.com',     // not 'url'
  link_title: 'Learn More',              // not 'title'
  source_page: '/home'                   // not 'page'
});
```

## User (People) Property Limits

User properties (via `Cryptique.people.*`) have stricter validation:

### Property Keys

| Constraint | Limit |
|-----------|-------|
| Maximum key length | 255 characters |
| Maximum keys per profile | 500 |

### Property Values

| Type | Constraint |
|------|------------|
| **String** | Max 255 bytes (UTF-8). Longer strings are truncated. |
| **Number** | Must be finite (no `NaN`, no `Infinity`) |
| **Boolean** | Allowed |
| **Array** | Allowed; elements validated recursively |
| **Object** | Max depth: 3 levels<br/>Max keys per level: 255<br/>Max total size: 256 KB |
| **Date string** | ISO 8601 format (e.g., `2025-01-15T00:00:00Z`) |

### Reserved User Properties

These property names are **reserved** and requests using them will be rejected:

```javascript
// ❌ These will cause errors
Cryptique.people.set({
  $name: 'John',        // Reserved
  $email: 'j@test.com', // Reserved
  $charges: 100,        // Reserved
  $phoneno: '555-1234'  // Reserved
});

// ✅ Use alternatives
Cryptique.people.set({
  name: 'John',
  email: 'j@test.com',
  phone: '555-1234'
});
```

## Identifier Limits

### distinct_id

| Constraint | Limit |
|-----------|-------|
| Maximum length | 500 characters |
| Format validation | None |

Generated format for anonymous users: `anon_{timestamp}_{userId}_{uuid}_{random}`

### user_id / session_id

| Constraint | Limit |
|-----------|-------|
| Maximum length | 255 characters |
| Format validation | None |

### identify_id

| Constraint | Limit |
|-----------|-------|
| Maximum length | 255 characters |
| Format validation | None |

```javascript
// ✅ Valid identifiers
Cryptique.identify('user_12345');
Cryptique.identify('a1b2c3d4-e5f6-7890-abcd-ef1234567890');
Cryptique.identify('0x742d35Cc6634C0532925a3b844Bc9e7595f5bB21');

// ❌ Too long (255+ chars)
Cryptique.identify('a'.repeat(300));
```

## Best Practices

### Event Properties

```javascript
// ✅ Good - reasonable payload
Cryptique.track('Trade Executed', {
  pair: 'ETH-USDC',
  side: 'buy',
  amount: 1.5,
  price: 1850.50,
  total: 2775.75
});

// ❌ Bad - overly large payload
Cryptique.track('Trade Executed', {
  orderBook: [...thousandsOfEntries],  // Too large
  fullTradeHistory: [...allTrades]      // Too much data
});
```

### User Properties

```javascript
// ✅ Good - within limits
Cryptique.people.set({
  plan: 'pro',
  totalTrades: 47,
  preferences: {
    theme: 'dark',
    notifications: true
  }
});

// ❌ Bad - exceeds limits
Cryptique.people.set({
  deeplyNested: {
    level1: {
      level2: {
        level3: {
          level4: 'too deep'  // Max depth is 3
        }
      }
    }
  }
});
```

### Identifiers

```javascript
// ✅ Good - reasonable length identifiers
Cryptique.identify('user_12345');
Cryptique.identify('0x742d35Cc6634C0532925a3b844Bc9e7595f5bB21');

// ❌ Bad - overly long
Cryptique.identify(JSON.stringify(entireUserObject));
```

## Error Handling

When limits are exceeded:

| Scenario | Behavior |
|----------|----------|
| Event name too long | Truncated or rejected |
| Reserved event property | Property silently removed |
| Reserved user property | Request rejected |
| User property too long | String truncated to 255 bytes |
| Too many user properties | Request may fail |
| Invalid number (NaN/Infinity) | Property rejected |

<Tip>
  Implement validation in your code before sending data to avoid silent truncation or rejection.
</Tip>
