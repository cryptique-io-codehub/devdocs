## Overview

When `autoEvents` is enabled (default), Cryptique automatically tracks user interactions without any additional code. This gives you immediate visibility into user behavior.

<Info>
  Auto-tracked events do **not** count against your custom event limits differently—they're tracked the same way as custom events.
</Info>

---

## Event Reference

Cryptique automatically tracks **10 event types**:

| Event | Description | Trigger |
|-------|-------------|---------|
| `page_view` | User views a page | Page load, SPA navigation |
| `element_click` | User clicks an element | Any click interaction |
| `rage_click` | User clicks rapidly (frustration signal) | 3+ clicks within 1 second |
| `dead_click` | User clicks non-interactive element | Click on element with no action |
| `page_scroll` | User scrolls the page | Scroll depth thresholds |
| `form_submit` | User submits a form | Form submission |
| `form_focus` | User focuses on form field | Input field focus |
| `media_play` | User plays media | Video/audio play |
| `media_pause` | User pauses media | Video/audio pause |
| `text_selection` | User selects text | Text highlight |

---

## Event Details

### page_view

Triggered when a user views a page. For SPAs, this fires on route changes.

**Auto-captured properties:**
- `page` - Current page path
- `title` - Page title
- `referrer` - Previous page URL
- `url` - Full page URL

---

### element_click

Triggered on any click interaction.

**Auto-captured properties:**
- `element_tag` - HTML tag name (e.g., `button`, `a`, `div`)
- `element_text` - Visible text content
- `element_classes` - CSS classes
- `element_id` - Element ID (if present)

---

### rage_click

A UX signal indicating user frustration. Triggered when a user clicks 3+ times within 1 second on the same area.

**Auto-captured properties:**
- `click_count` - Number of rapid clicks
- `element_tag` - Target element tag
- `element_text` - Target element text

<Tip>
  High rage click rates often indicate UI issues—buttons that don't respond, confusing interfaces, or performance problems.
</Tip>

---

### dead_click

Triggered when a user clicks on a non-interactive element (no resulting action). Indicates user expectation mismatch.

**Auto-captured properties:**
- `element_tag` - Clicked element tag
- `element_text` - Clicked element text

---

### page_scroll

Tracks scroll depth to understand content engagement.

**Auto-captured properties:**
- `scroll_depth` - Percentage scrolled (25%, 50%, 75%, 100%)
- `page` - Current page

---

### form_submit

Triggered when a form is submitted.

**Auto-captured properties:**
- `form_id` - Form element ID
- `form_name` - Form name attribute
- `form_action` - Form action URL

<Note>
  Cryptique does **not** capture form field values automatically to protect user privacy. Use custom events if you need to track specific (non-sensitive) field data.
</Note>

---

### form_focus

Triggered when a user focuses on a form input.

**Auto-captured properties:**
- `field_type` - Input type (text, email, etc.)
- `field_name` - Input name attribute
- `form_id` - Parent form ID

---

### media_play / media_pause

Tracks video and audio engagement.

**Auto-captured properties:**
- `media_type` - `video` or `audio`
- `media_src` - Media source URL
- `media_duration` - Total duration
- `current_time` - Playback position

---

### text_selection

Triggered when a user highlights text.

**Auto-captured properties:**
- `selected_text` - The highlighted text (truncated for privacy)
- `selection_length` - Character count

---

## Disabling Auto-Events

### At Initialization

```javascript
Cryptique.init({
  siteId: 'your-site-id',
  autoEvents: true,
  disabledEvents: ['page_scroll', 'text_selection', 'media_play', 'media_pause']
});
```

### Via CDN

```html
<script 
  src="https://cdn.cryptique.io/scripts/analytics/1.0.1/cryptique.script.min.js" 
  site-id="your-site-id" 
  auto-events="true"
  auto-events-disabled-events="page_scroll,text_selection,media_play,media_pause">
</script>
```

### At Runtime

```javascript
// Disable specific events
Cryptique.disableAutoEvent('page_scroll');
Cryptique.disableAutoEvent(['text_selection', 'media_play']);

// Re-enable events
Cryptique.enableAutoEvent('page_scroll');

// Replace entire disabled list
Cryptique.setAutoEventsDisabledEvents(['rage_click', 'dead_click']);
```

---

## Disabling by Path

Exclude specific URL paths from all auto-tracking:

```javascript
Cryptique.init({
  siteId: 'your-site-id',
  disabledPaths: [
    '/admin',
    '/internal',
    '/checkout'  // Maybe you want manual control here
  ]
});
```

Via CDN:

```html
<script 
  src="https://cdn.cryptique.io/scripts/analytics/1.0.1/cryptique.script.min.js" 
  site-id="your-site-id" 
  auto-events="true"
  auto-events-disabled-paths="/admin,/internal,/checkout">
</script>
```

## Wallet & Session Data

In addition to auto-tracked events, Cryptique captures **session-level data**:

| Data | Description |
|------|-------------|
| **Session ID** | Unique session identifier |
| **Session Duration** | Time spent in session |
| **Page Count** | Pages viewed in session |
| **Wallet Connected** | Boolean - is wallet connected |
| **Wallet Address** | Connected wallet address (if any) |
| **Chain ID** | Connected chain (if wallet connected) |
| **Wallet Type** | Detected wallet provider |

<Note>
  Wallet connection status is tracked as session data, not as separate `wallet_connected` / `wallet_disconnected` events. This data is sent with session payloads.
</Note>

## On-Chain Transaction Tracking

When you add smart contracts in your dashboard, **all transactions are automatically indexed**. This happens server-side—no SDK configuration needed.

Tracked transaction data includes:
- Transaction hash
- From/to addresses
- Value transferred
- Gas used
- Contract method called
- Block number & timestamp
- Transaction status

## Recommended Configuration

<Tabs>
  <Tab title="Most Apps">
    Track everything except verbose events:
    
    ```javascript
    Cryptique.init({
      siteId: 'your-site-id',
      autoEvents: true,
      disabledEvents: ['text_selection']
    });
    ```
  </Tab>
  
  <Tab title="Performance-Focused">
    Minimal tracking for lightweight footprint:
    
    ```javascript
    Cryptique.init({
      siteId: 'your-site-id',
      autoEvents: true,
      disabledEvents: [
        'page_scroll',
        'text_selection',
        'media_play',
        'media_pause',
        'form_focus'
      ]
    });
    ```
  </Tab>
  
  <Tab title="Manual Only">
    Disable all auto-tracking:
    
    ```javascript
    Cryptique.init({
      siteId: 'your-site-id',
      autoEvents: false
    });
    ```
  </Tab>
</Tabs>
