## Overview

Cryptique can be deployed through any tag manager that supports custom HTML or JavaScript. The setup process is similar across platforms: add the Cryptique script as a custom tag.

## Universal Script

Use this script in any tag manager:

```html
<script>
  var script = document.createElement('script');
  script.src = 'https://cdn.cryptique.io/scripts/analytics/1.0.1/cryptique.script.min.js';  
  script.setAttribute('site-id', 'YOUR_SITE_ID');
  script.setAttribute('auto-events', 'true');
  document.head.appendChild(script);
</script>
```

Replace `YOUR_SITE_ID` with your Site ID from the Cryptique dashboard.

### Device-Mode Integration

Add Cryptique as a custom script in your Segment snippet:

```html
<script>
  // After Segment loads
  analytics.ready(function() {
    var script = document.createElement('script');
    script.src = 'https://cdn.cryptique.io/scripts/analytics/1.0.1/cryptique.script.min.js';  
    script.setAttribute('site-id', 'YOUR_SITE_ID');
    script.setAttribute('auto-events', 'true');
    document.head.appendChild(script);
  });
</script>
```

### Forwarding Segment Events

Forward Segment track calls to Cryptique:

```javascript
analytics.on('track', function(event, properties) {
  if (window.cryptique) {
    window.cryptique.track(event, properties);
  }
});

analytics.on('identify', function(userId, traits) {
  if (window.cryptique) {
    window.cryptique.identify(userId, traits);
  }
});
```

Forward Tealium events to Cryptique:


## Generic Setup Pattern

For any tag manager not listed above, follow this pattern:

### Requirements

Your tag manager needs to support:
- Custom HTML or JavaScript tags
- Page-level or global triggers
- Variable substitution (optional but helpful)

### Steps

1. **Find custom tag option**: Look for "Custom HTML", "Custom Code", "JavaScript Tag", or similar
2. **Add the script**: Paste the Cryptique initialization script
3. **Set trigger**: Configure to fire on all pages (or specific pages)
4. **Test**: Use preview/debug mode to verify
5. **Publish**: Deploy to production

### Script Template

```html
<script>
  (function() {
    // Configuration
    var config = {
      siteId: 'YOUR_SITE_ID',
      autoEvents: true,
      disabledPaths: '',      // Optional: '/admin,/internal'
      disabledEvents: ''      // Optional: 'copy,input_change'
    };
    
    // Create and configure script
    var script = document.createElement('script');
    script.src = 'https://cdn.cryptique.io/scripts/analytics/1.0.1/cryptique.script.min.js';
    script.setAttribute('site-id', config.siteId);
    script.setAttribute('auto-events', config.autoEvents);
    
    if (config.disabledPaths) {
      script.setAttribute('auto-events-disabled-paths', config.disabledPaths);
    }
    if (config.disabledEvents) {
      script.setAttribute('auto-events-disabled-events', config.disabledEvents);
    }
    
    // Load script
    document.head.appendChild(script);
  })();
</script>
```

## Event Forwarding

### From Tag Manager to Cryptique

Most tag managers can forward events to Cryptique:

```javascript
// Generic event forwarding pattern
function forwardToCryptique(eventName, properties) {
  if (window.cryptique) {
    window.cryptique.track(eventName, properties);
  } else {
    // Queue for later if Cryptique hasn't loaded
    window.cryptiqueQueue = window.cryptiqueQueue || [];
    window.cryptiqueQueue.push({ event: eventName, props: properties });
  }
}

// Process queue when Cryptique loads
document.addEventListener('cryptique:ready', function() {
  if (window.cryptiqueQueue) {
    window.cryptiqueQueue.forEach(function(item) {
      window.cryptique.track(item.event, item.props);
    });
    window.cryptiqueQueue = [];
  }
});
```

### User Identification

Forward user identity when available:

```javascript
function identifyInCryptique(userId, traits) {
  if (window.cryptique) {
    window.cryptique.identify(userId, traits);
  }
}

// Call when user logs in or is identified
identifyInCryptique('user_123', {
  email: 'user@example.com',
  plan: 'pro'
});
```

## Verification

### Check Script Loading

In browser console:

```javascript
// Verify Cryptique is loaded
console.log('Cryptique loaded:', !!window.cryptique);

// Check site ID
if (window.cryptique) {
  console.log('Site ID:', window.cryptique.getSiteId());
}
```

### Check Network Requests

Look for requests to:
- `cdn.cryptique.io` - Script loading
- `api.cryptique.io` - Event tracking

### Check Cryptique Dashboard

1. Go to Live Events in Cryptique
2. Perform actions on your site
3. Verify events appear within 60 seconds

## Troubleshooting

<AccordionGroup>
  <Accordion title="Script not loading">
    - Check tag manager container is on the page
    - Verify trigger conditions are met
    - Look for JavaScript errors in console
    - Check if ad blockers are interfering
  </Accordion>
  
  <Accordion title="Events not tracking">
    - Verify Site ID is correct
    - Check domain whitelist in Cryptique settings
    - Ensure script loaded before tracking calls
    - Check network tab for API errors
  </Accordion>
  
  <Accordion title="Duplicate events">
    - Ensure Cryptique only loads once
    - Check for multiple tag manager containers
    - Don't combine tag manager with direct installation
  </Accordion>
</AccordionGroup>
